# 安全审查清单

## 输入验证
- 所有用户输入是否校验（类型、长度、格式）
- URL 参数是否编码
- 文件上传是否校验类型和大小

## 认证与授权
- 密码是否哈希存储（bcrypt/argon2）
- Token 是否设置合理过期时间
- 是否实现最小权限原则
- 敏感操作是否二次验证

## 注入防护
- SQL: 参数化查询，禁止拼接
- XSS: 输出转义，CSP 头
- Command: 禁止 shell 拼接，使用 execFile
- LDAP: 参数化过滤器
- Path: 路径标准化，限制范围

## 数据保护
- 敏感数据是否加密存储
- 传输是否强制 HTTPS
- 日志是否脱敏（不记录密钥、token）
- 错误信息是否泄露栈详情

## 依赖安全
- 是否定期审计依赖（npm audit）
- 是否锁定版本（lockfile）
- 是否使用最小依赖原则
- 是否关注 CVE 公告
