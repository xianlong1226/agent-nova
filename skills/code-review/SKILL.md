# Code Review Skill

## 能力
审查代码质量，发现 bug、安全漏洞、性能问题、可维护性问题

## 何时使用
- 用户要求 review 代码
- 提交 PR 前
- 代码重构后

## 审查维度
1. **正确性**：逻辑漏洞、边界情况、类型安全
2. **安全性**：注入攻击、敏感信息泄露、权限问题
3. **性能**：N+1 查询、内存泄漏、不必要的计算
4. **可维护性**：命名规范、代码结构、注释质量
5. **最佳实践**：设计模式、SOLID 原则、DRY

## 输出格式
```
## 审查报告

### 🔴 严重问题
- [文件:行号] 问题描述 + 修复建议

### 🟡 建议优化
- [文件:行号] 优化建议

### 🟢 做得好
- 值得肯定的地方
```
